Aml.Editor.Plugin.Base Namespace

AutomationML Editor
The Aml.Editor.Plugin.Base name space contains abstract Base Classes for AutomationML Editor PlugIn Developers which implement some basic Functionality for PlugIns. Example Implementations, that may be used as a starting point for PlugIn Development are available on the Aml.Editor.Plugin Project side for Developers.
Classes
  ClassDescription
PluginBase
PluginBase is an abstract base class for PlugIn Implementation which is a basic implementation of the IAMLEditorPlugin PlugIn Interface
Remarks
To get a PlugIns loaded into the AutomationML Editor at startup the DLLs have to be put in the PlugIns Folder in the Execution Directory of the Editor. Only the AutomationML Editor Version 4 and higher supports 3d Party PlugIns. Every PlugIn shall use the same version of the Aml.Engine as the used Aml.Editor version.
